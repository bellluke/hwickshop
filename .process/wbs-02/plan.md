# WBS-02 실행 계획: UI/UX 개선 — "AI 프로토타입" 탈피

## 1. 문제 진단

### 1.1 "AI 프로토타입처럼 보인다"의 구체적 의미

AI가 생성한 UI가 공유하는 패턴("AI slop")을 현재 사이트가 거의 그대로 따르고 있다.

| AI slop 패턴 | 현재 사이트의 해당 사항 |
|---|---|
| Tailwind 기본 색상표 직접 사용 | 8개 카테고리 컬러가 전부 Tailwind 기본 팔레트(`amber-400`, `emerald-500` 등) |
| 균일한 간격·라운드·그림자 | `rounded-2xl` + `shadow-card` 2종만 전 컴포넌트 반복 |
| 제네릭 카드 레이아웃 | 모든 카드가 동일한 border + shadow + hover 패턴 |
| 과도한 이모지·컬러 배지 | 8개 색상 배지가 한 화면에 동시 표시 |
| 단조로운 호버 효과 | `-translate-y-px` + 그림자 변경만 반복 |
| 균등한 그리드 | 2×4 동일 크기 카드, 시각적 계층 없음 |
| 모든 페이지 동일 구조 | content-width(720px) 하나로 전 페이지 통일 |

### 1.2 핵심 원인: "의도적 선택"의 부재

- **색상**: 왜 이 색인지 이유 없이 Tailwind 기본값 채택
- **간격**: 왜 이 여백인지 이유 없이 py-16, pb-12 같은 기본 배수 사용
- **컴포넌트**: 왜 이 형태인지 이유 없이 border + rounded + shadow 공식 반복
- **타이포**: 왜 이 굵기·행간인지 이유 없이 h1=800, h2=700 기계적 할당

## 3. 단계별 상세 계획

### Phase 1: 디자인 토큰 리디자인 (기반)

#### 1-1. 색상 체계 리디자인

**현재 문제:**
- 8개 카테고리 색상이 모두 Tailwind 기본 팔레트의 100% 채도
- 메인 오렌지 외에 8개 색이 동시에 경쟁 → 시각적 혼란
- 배지에 카테고리 색상을 직접 사용하여 "색상 테스트" 같은 느낌

**변경 방향:**
```
[현재] 8색 고채도 — 모든 카테고리가 고유 색상으로 강하게 표현
[목표] 1+2 체계 — 메인 오렌지 + 뉴트럴(stone) + 포인트(1색만 상황별 사용)
```

**구체적 변경:**
1. 카테고리 색상을 CSS 변수에서 제거하지는 않되, 사용 범위를 극적으로 축소
   - 배지: 컬러 배경 제거 → 텍스트만 stone 컬러 + 좌측 얇은 세로선(카테고리색)
   - 카테고리 페이지: 페이지 상단 액센트 라인(2px)으로만 사용
   - 홈: 카테고리 그리드에서 이모지를 제거하고 텍스트 중심으로 변경
2. 카테고리 색상 채도를 낮춤 (HSL 기준 saturation 60~70%로 조정)
3. 배경색 `#FFFBF5` 유지 (이건 좋은 선택)
4. 그라디언트 전면 제거 (시즌 배너의 `from-amber-50 to-orange-50` 등)

#### 1-2. 타이포그래피 위계 재설계

**현재 문제:**
- h1=800, h2=700, h3=700 → 기계적 weight 할당
- 모든 본문 line-height: 1.75 고정
- 가격 전부 monospace → 과도한 강조

**변경 방향:**
```
[현재] 크기와 굵기만으로 위계 → 납작한 위계
[목표] 크기 + 굵기 + 색상 + 자간 + 행간의 복합 위계 → 풍부한 위계
```

**구체적 변경:**
1. h1: `font-weight: 800` → `700`, 자간 `-0.03em`으로 촘촘하게, 색상 유지
2. h2: `font-weight: 700` → `600`, h2 아래 border-bottom 제거 → 여백으로 구분
3. h3: `font-weight: 700` → `600`, 크기를 h2와 더 차별화
4. 본문 line-height: `1.75` → `1.7` (미세 조정)
5. 보조 텍스트(캡션, 메타)에 `text-stone-400` 적용하여 주/보조 대비 강화
6. 가격: monospace 사용 유지하되, `font-weight: 700` → `600`, 사용처 선별
7. 글 페이지 본문 max-width: `720px` → `680px` (더 좁게, 가독성 향상)

#### 1-3. 그림자·라운드·보더 시스템 재정의

**현재 문제:**
- 그림자 2종 (`shadow-card`, `shadow-card-hover`)로 모든 컴포넌트 처리
- `rounded-2xl`(16px) 남발 → 프로토타입 느낌
- 모든 카드에 `border border-stone-200` 동일 적용

**변경 방향:**
```
[현재] 그림자+보더+라운드 삼중 장식 → 과도한 카드 느낌
[목표] 요소별 최소한의 구분 → 콘텐츠가 돋보이는 절제
```

**구체적 변경:**
1. 그림자 4단계 정의:
   - `--shadow-xs`: `0 1px 2px rgba(0,0,0,0.04)` (미세한 구분)
   - `--shadow-sm`: `0 2px 6px rgba(0,0,0,0.06)` (기본 카드)
   - `--shadow-md`: `0 4px 12px rgba(0,0,0,0.08)` (강조 카드)
   - `--shadow-lg`: `0 8px 24px rgba(0,0,0,0.1)` (모달/오버레이)
2. 라운드 규칙:
   - 카드: `rounded-xl`(12px) → 더 절제된 느낌
   - 버튼: `rounded-lg`(8px) 유지
   - 배지/태그: `rounded-md`(6px) → 과도한 pill 형태 제거
   - 입력: `rounded-lg`(8px)
3. 보더 축소:
   - 카드: `border border-stone-200` → `border border-stone-100` (더 연하게)
   - 일부 카드는 보더 제거하고 배경색 차이만으로 구분

### Phase 2: 페이지 레이아웃 리디자인

#### 2-1. 홈페이지 리디자인

**현재 문제:**
- 히어로 → 카테고리 그리드 → 인기글 → 시즌배너 = 4섹션이 균등 간격으로 나열
- 모든 섹션이 동일한 `content-width`(720px)
- 8개 카테고리가 동일 크기 카드로 균등 배치 → "AI 대시보드"

**변경 방향:**
```
[현재] 4개 섹션의 기계적 나열
[목표] 시선 흐름이 있는 스토리텔링 페이지
```

**구체적 변경:**
1. 히어로: 텍스트 중심으로 유지하되, 아래 여백을 크게 확보 (py-16 → py-20)
   - "뭘 시작하세요?" 텍스트의 크기를 줄여 보조 역할로 전환
2. 카테고리: 균등 그리드 대체
   - 이모지 제거, 텍스트만 남김
   - 2×4 그리드 대신 리스트 형태 또는 비균등 그리드 검토
   - 각 카테고리에 글 개수 표시 (예: "자취 · 5편")
3. 인기글: 카드 스타일 단순화
   - 컬러 배지 제거 → 카테고리명을 작은 텍스트로 표시
   - 카드 호버: 그림자 변경 대신 배경색 미세 변화
4. 시즌 배너: 그라디언트 제거
   - 단색 배경(stone-50) + 텍스트만으로 단순화
   - 또는 배너 자체를 제거하고 인기글 섹션에 통합

#### 2-2. 글 페이지 prose 스타일 리디자인

**현재 문제:**
- h2 아래 border-bottom이 모든 h2에 기계적 적용
- blockquote가 TipBox와 동일 스타일 (주황 왼쪽 보더 + 노란 배경)
- 모든 문단에 동일한 margin-bottom
- 콘텐츠의 따뜻한 톤 vs CSS의 기계적 톤 불일치

**변경 방향:**
```
[현재] 마크다운-to-HTML 자동 변환의 기본 스타일
[목표] 잘 편집된 매거진/블로그의 읽기 경험
```

**구체적 변경:**
1. h2: border-bottom 제거, 상단 여백을 키워 자연스러운 섹션 구분 (margin-top: 3rem)
2. h3: 상단 여백 조정 (margin-top: 2rem), 색상을 text-stone-700으로 미세 변화
3. blockquote: 스타일 재설계
   - 배경색 제거 → 좌측 보더(2px, stone-300) + 들여쓰기만
   - italic 제거 → 일반 본문 스타일 유지
   - 인용 출처는 작은 텍스트로 아래 배치
4. 리스트(ul/ol): 불릿 스타일 커스텀, 간격 미세 조정
5. 테이블: 배경 줄무늬 제거, 세로선 제거하고 가로선만 (stone-100)
6. 글 상단 메타(카테고리·읽는 시간·링크 수): 배지 스타일 단순화

#### 2-3. 카테고리 페이지 리디자인

**변경 방향:**
- 카테고리 타이틀에 색상 사용 축소 → 텍스트만으로 표현
- 글 카드의 컬러 배지 제거
- 카드 간 간격에 리듬 부여

### Phase 3: 컴포넌트 리디자인

#### 3-1. ProductCard

**현재 문제:**
- 7가지 Tailwind 클래스 중첩 → 과도한 장식
- 모든 ProductCard가 동일한 구조·크기·호버
- CTA 버튼이 "쿠팡에서 보기" + 장바구니 아이콘 → 광고 느낌

**변경 방향:**
```
[현재] 독립된 "상품 카드" 컴포넌트 (쇼핑몰의 상품 목록 같은)
[목표] 글 흐름에 자연스럽게 녹아드는 추천 블록
```

**구체적 변경:**
1. 외곽: `rounded-2xl border shadow` → `border-b border-stone-100` (하단 구분선만)
   또는 배경색 차이(`bg-stone-50`)로만 본문과 구분
2. 이미지: 있으면 좌측 작은 썸네일(60×60), 없으면 생략
3. 상품명 + 가격: 같은 줄에 배치하여 공간 절약
4. CTA: "쿠팡에서 보기" 풀버튼 → 텍스트 링크 + 화살표 ("→ 쿠팡에서 보기")
5. 한줄 설명: 위치를 상품명 바로 아래로 이동
6. 별점: 제거 또는 축소 (별점이 신뢰도보다는 광고 느낌을 줌)

#### 3-2. Checklist

**현재 문제:**
- 카드 형태의 독립된 박스 → 본문과 분리된 "위젯" 느낌
- 하단 합계 바가 앱의 쇼핑 카트 같음

**변경 방향:**
```
[현재] 독립된 "체크리스트 위젯"
[목표] 글 흐름의 일부인 인터랙티브 목록
```

**구체적 변경:**
1. 외곽 카드 제거 → 본문에 자연스럽게 이어지는 목록
2. 상단 제목: 카드 헤더 스타일 → 일반 h4 또는 작은 레이블
3. 하단 합계: 고정 바 → 우측 하단에 작은 텍스트 ("선택 3/8 · 합계 129,000원")
4. 체크박스: 기본 체크박스 → 좀 더 커스텀된 스타일 (stroke 애니메이션 등)
5. 가격: monospace 유지하되 크기 축소
6. 카테고리 구분: `uppercase tracking-wide` 제거 → 일반 굵은 텍스트

#### 3-3. BudgetBox, TipBox, DontBuy

**BudgetBox 변경:**
- 프로그레스 바의 둥근 모서리 축소 (`rounded-full` → `rounded-sm`)
- 3가지 예산 단계의 색상: 3색(emerald, orange, violet) → 1색(primary)의 투명도 3단계로
- 외곽: 카드 스타일 → 간단한 상단 구분선만

**TipBox 변경:**
- `border-l-4 bg-amber-50 italic` 제거
- 대신: 들여쓰기 + 좌측 얇은 선(1px, stone-200) + 일반 텍스트
- 출처(author): "—" 접두어 유지, 색상만 stone-400으로

**DontBuy 변경:**
- `border-l-4 border-red-400 bg-stone-100` → 배경 제거
- 대신: 제목에 작은 ✕ 아이콘 + 일반 리스트 형태
- 붉은 계열 색상 축소 (경고가 아닌 "참고"의 톤)

#### 3-4. Header, Footer, 네비게이션

**Header 변경:**
- 로고 "휙": 현재 스타일 유지 (Space Grotesk)
- 모바일 메뉴: 이모지 제거, 텍스트만
- 하단 보더: `border-b border-stone-100`으로 더 연하게
- 데스크탑: 검색 아이콘 제거 (현재 기능 없음, 장식성 제거)

**Footer 변경:**
- 배경: `bg-stone-50` → `bg-transparent` + 상단 border만
- 텍스트 정렬: 중앙 → 좌측 (더 자연스러운 읽기)
- 디스클레이머: 스타일 단순화

### Phase 4: 인터랙션 및 반응형

#### 4-1. 호버·전환·마이크로인터랙션

**변경:**
1. 카드 호버: `hover:-translate-y-px` 제거 → `hover:bg-stone-50` (배경색 변화만)
2. transition duration: `duration-200` 통일 → 용도별 차등
   - 색상 변화: `duration-150` (빠르게)
   - 레이아웃 변화: `duration-300` (부드럽게)
3. 링크 호버: underline 대신 color 변화 (stone-500 → stone-900)
4. 체크리스트 체크: 체크 시 부드러운 줄긋기 + opacity 변화

#### 4-2. 모바일 세밀 조정

**변경:**
1. breakpoint 추가: 480px (small mobile)
2. 모바일 여백: `padding-inline: 1rem` → `1.25rem` (미세하게 넓게)
3. 카테고리 그리드: 모바일 2열 → 1열 리스트 형태 검토
4. ProductCard: 모바일에서 이미지 위치 조정

## 4. 작업 순서 및 의존성

```
Phase 1 (기반) ─→ Phase 2 (페이지) ─→ Phase 3 (컴포넌트) ─→ Phase 4 (인터랙션) ─→ Phase 5 (QA)
  1-1 색상          2-1 홈             3-1 ProductCard      4-1 호버            5-1 통합QA
  1-2 타이포        2-2 글 페이지      3-2 Checklist        4-2 모바일
  1-3 그림자/라운드  2-3 카테고리       3-3 기타 컴포넌트
                                       3-4 Header/Footer
```

Phase 1은 디자인 토큰 변경이므로 모든 후속 작업에 영향. 반드시 먼저 완료.
Phase 2~3는 순서 유연하나, Phase 2 먼저 하는 것이 전체 레이아웃 확인 후 컴포넌트 세밀 조정에 유리.
